2 — Job Discovery & Search

sequenceDiagram
    participant FE as Frontend JobPage
    participant JS as jobs-search
    participant FC as Firecrawl API
    participant GM as Gemini API
    participant DB as Supabase DB

    FE->>DB: Read job_source_settings
    DB-->>FE: Allowed domains + preferences

    FE->>JS: POST /functions/v1/jobs-search
    JS->>DB: Load job_source_settings
    DB-->>JS: Allowed domains list

    JS->>FC: POST /v1/search - Google-style query
    FC-->>JS: Search results with URLs

    loop For each result
        JS->>FC: POST /v1/scrape job URL
        FC-->>JS: Raw page content
        JS->>GM: generateContent - extract title, company, salary, reqs
        GM-->>JS: Extracted job JSON
    end

    JS->>DB: INSERT INTO jobs per-user
    JS-->>FE: Job list response

    FE->>FE: Display job cards in queue

    rect rgba(100,200,100,0.1)
    Note over FE,GM: URL-based Intake Alternative
    FE->>JS: POST /functions/v1/intake-job-url
    JS->>FC: Firecrawl extract
    FC-->>JS: Page content
    JS->>GM: Gemini parse
    GM-->>JS: Structured job
    JS->>DB: UPSERT jobs
    JS-->>FE: Extracted job
    end

3 — Job Evaluation & Match Scoring

sequenceDiagram
    participant FE as Frontend JobPage
    participant MS as calculate-match-score
    participant EV as evaluate-job-fit
    participant GM as Gemini API
    participant DB as Supabase DB

    Note over FE: Job list loaded from jobs table

    FE->>MS: POST /functions/v1/calculate-match-score
    MS->>DB: Load profile + skills + experiences
    DB-->>MS: User profile data
    MS->>DB: Load jobs by IDs
    DB-->>MS: Job details
    MS->>GM: generateContent - batch match scoring
    GM-->>MS: Scores + reasoning per job
    MS-->>FE: Match scores with breakdowns

    FE->>FE: Sort and badge jobs by match %

    rect rgba(200,100,100,0.1)
    Note over FE,DB: Deep Evaluation per job
    FE->>EV: POST /functions/v1/evaluate-job-fit
    EV->>DB: Load profile + candidate_memory + parsed_resume
    DB-->>EV: Full candidate context
    EV->>DB: Load job details
    DB-->>EV: Job record
    EV->>GM: generateContent - detailed fit analysis
    GM-->>EV: Structured evaluation report
    EV->>DB: INSERT job_evaluations + UPDATE jobs.evaluation_summary
    EV-->>FE: Evaluation report
    FE->>FE: Show fit report card
    end

4 — Application & Auto-Apply via Skyvern

sequenceDiagram
    participant FE as Frontend JobPage
    participant AP as apply-to-jobs
    participant SK as Skyvern API
    participant WH as skyvern-webhook
    participant DB as Supabase DB
    participant AT as Frontend ApplicationPage

    FE->>AP: POST /functions/v1/apply-to-jobs

    AP->>DB: Load job details + profile
    DB-->>AP: Job record + candidate data

    AP->>DB: Load source_credentials encrypted
    DB-->>AP: Per-domain credentials
    AP->>AP: Decrypt credentials

    AP->>SK: POST /v1/run/workflows
    SK-->>AP: run_id

    AP->>DB: INSERT applications status in_progress
    AP-->>FE: applicationId + run_id

    FE->>FE: Show Applying status

    loop Poll status
        FE->>SK: GET /functions/v1/get-run/run_id
        SK-->>FE: Current run status
    end

    SK->>WH: POST webhook with run_id + status
    WH->>DB: UPDATE applications SET status + recording_url

    rect rgba(100,200,100,0.1)
    Note over WH,DB: On successful apply
    WH->>DB: DELETE FROM jobs WHERE applied
    end

    AT->>DB: SELECT FROM applications
    DB-->>AT: Application records
    AT->>AT: Render Kanban tracker

6 — AI Chat Assistant - Ask & Agent Modes

sequenceDiagram
    participant FE as Frontend ChatPage
    participant AC as ai-chat Edge Function
    participant DB as Supabase DB
    participant EF as Other Edge Functions
    participant GM as Gemini API

    FE->>AC: POST /functions/v1/ai-chat mode agent + messages

    AC->>DB: fetchUserContext - profile, jobs, apps, resumes, credits, memory
    DB-->>AC: UserContext object

    AC->>GM: generateContent with messages + functionDeclarations
    GM-->>AC: Response with functionCall parts

    loop Up to 5 tool turns
        AC-->>FE: SSE tool_call event

        alt DB tool - list_applications or list_recent_jobs
            AC->>DB: Query with user auth token
            DB-->>AC: Query results
        else run_job_search
            AC->>EF: POST /functions/v1/jobs-search
            EF-->>AC: Search results
        else evaluate_job_fit
            AC->>EF: POST /functions/v1/evaluate-job-fit
            EF-->>AC: Evaluation report
        else apply_to_job
            AC->>EF: POST /functions/v1/apply-to-jobs
            EF-->>AC: Application status
        else intake_job_url
            AC->>EF: POST /functions/v1/intake-job-url
            EF-->>AC: Extracted job
        else generate_cover_letter
            AC->>EF: POST /functions/v1/generate-cover-letter
            EF-->>AC: Cover letter
        end

        AC->>GM: generateContent with functionResponse
        GM-->>AC: Next response - text or more calls
    end

    AC-->>FE: SSE message events - streamed text
    AC-->>FE: SSE done event

    FE->>DB: UPDATE chat_sessions messages + response_id

    rect rgba(200,100,200,0.1)
    Note over FE,GM: First message in session
    FE->>EF: POST /functions/v1/generate-title
    EF->>GM: generateContent - short title
    GM-->>EF: Title text
    EF-->>FE: Session title
    FE->>DB: UPDATE chat_sessions.title
    end

7 — Background Jobs & Cron Ingestion

sequenceDiagram
    participant CR as jobs-cron Scheduled
    participant RM as Remotive API
    participant RO as RemoteOK API
    participant AN as ArbeitNow API
    participant FC as Firecrawl API
    participant GM as Gemini API
    participant DB as Supabase DB

    Note over CR: Runs on schedule with service-role

    CR->>RM: GET /api/remote-jobs
    RM-->>CR: Job listings JSON
    CR->>RO: GET /api
    RO-->>CR: Job listings JSON
    CR->>AN: GET /api/jobs
    AN-->>CR: Job listings JSON

    CR->>CR: Normalize + deduplicate

    loop New or updated listings
        CR->>FC: POST /v1/scrape if needed
        FC-->>CR: Page content
        CR->>GM: generateContent - extract structured fields
        GM-->>CR: Structured job JSON
        CR->>CR: parseSalaryRangeToMinMax
    end

    CR->>DB: UPSERT INTO jobs and job_listings batch
    Note over CR,DB: Also triggers process-job-search for queued user searches

8 — Payments & Subscription Gating

sequenceDiagram
    participant FE as Frontend PricingPage
    participant IP as init-payment
    participant FX as Exchange Rate API
    participant PS as Paystack API
    participant WH as paystack-webhook
    participant DB as Supabase DB

    FE->>IP: POST /functions/v1/init-payment
    IP->>FX: GET open.er-api.com USD to NGN
    FX-->>IP: Exchange rate
    IP->>IP: Calculate amount in NGN
    IP->>PS: POST /transaction/initialize
    PS-->>IP: authorization_url + reference
    IP-->>FE: Redirect URL
    FE->>FE: Redirect to Paystack checkout

    PS->>WH: POST webhook event
    WH->>PS: GET /transaction/verify/ref
    PS-->>WH: Verified transaction
    WH->>DB: UPDATE user_subscriptions + INSERT user_credits
    WH-->>PS: 200 OK

    Note over FE,DB: Tier gates: Pro = AI Chat + Agent | Basics+ = Auto-apply + Evaluation | Free = Limited
